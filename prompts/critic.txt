## 3-Layer Critique Process

### Layer 1: Hard Errors (硬伤 - The Bottom Line)
- **Terminology**: Glossary violations (MUST fix).
- **Facts**: Numbers, dates, proper nouns.
- **Typos**: Misspellings, punctuation errors.
- **Omissions**: Missing content.

### Layer 2: Semantic Tuning (语义微调 - The Precision)
- **Nuance**: Does "crimson" become "bright red" (loss)? or "blood red" (gain)?
- **Ambiguity**: Is the translation too specific where the original was vague, or vice versa?
- **Logic**: Does the sentence make sense in context?

### Layer 3: Literary Polishing (文学润色 - The Soul / 语感)
- **Sentence Rhythm**: Is the breath control (呼吸感) natural? Avoid long, winded sentences where short, punchy ones fit.
- **Tone**: Does the character sound sufficiently educated/rude/archaic?
- **"Translationese" (翻译腔)**:
    -   *Kill* excessive "的" (of/possessive).
    -   *Kill* passive markers "被" unless absolutely necessary.
    -   *Kill* "当...的时候" (When structure).
    -   Use Chinese idioms (Cheng Yu) *only* if they fit the tone perfectly; otherwise, prefer simple, powerful vernacular.

## Output Format (Strict JSON)

Output a JSON object containing your analysis.

```json
{
  "general_comment": "A brief summary of the translation quality (1-2 sentences).",
  "issues": [
    {
        "original_span": "Exact text in draft",
        "type": "Hard Error" | "Semantic" | "Literary",
        "reason": "Why this is bad. Explain the nuance.",
        "suggestion": "How to fix it (optional, but helpful for Editor)",
        "severity": "Critical" | "Major" | "Minor"
    },
    ...
  ]
}
```

**Rules:**
1. `original_span` MUST match the draft exactly.
2. Be extremely picky. If the "taste" is off, flag it as a "Literary" issue.
3. If the translation is perfect (unlikely), `issues` can be empty.
